# Contributing to numerical-fracture-datasets

Thank you for your interest in contributing! This document explains how to add
a new dataset series or improve existing ones.

---

## Adding a new dataset series

1. **Fork** the repository and create a feature branch:
   ```bash
   git checkout -b add/<approach>/<series-name>
   ```

2. **Create a directory** under the appropriate approach:
   ```
   datasets/<approach>/<series-name>/
   ```
   If the approach does not have a folder yet, create it and add a `README.md`
   describing the method.

3. **Add a `README.md`** inside the series directory. It must include:
   - Short description of the problem and model
   - Software / code used to generate the data
   - Key simulation parameters (as a table or in a `params.json` / `params.yaml`)
   - File inventory (filename, description, units)
   - Provenance: who generated the data and when
   - Citation or reference if the data accompanies a paper

4. **Add data files.** Large binary files are tracked with Git-LFS. Make sure
   Git-LFS is installed (`git lfs install`) before you stage these files:
   ```bash
   git add datasets/<approach>/<series-name>/
   git commit -m "Add dataset: <series-name>"
   ```
   Git will automatically apply the LFS filters defined in `.gitattributes`.

5. **Open a pull request** against `main`. Describe what the dataset contains
   and link any relevant paper or code repository.

## Dataset README template

```markdown
# <Series Name>

## Description
<Brief description of the fracture problem and numerical model>

## Software
- Code: <e.g. FEniCSx v0.7>
- Language: Python 3.x

## Parameters
| Parameter | Value | Units |
|---|---|---|
| Domain size | 1 × 1 | m |
| Mesh size | ... | m |
| ... | ... | ... |

## Files
| File | Description |
|---|---|
| `results.h5` | Full displacement and damage fields |
| `load_disp.csv` | Load-displacement curve |

## Provenance
Generated by <author> on <date>.

## Citation
If you use this dataset, please cite: <reference or DOI>
```

---

## Code style and scripts

If you include a loading script (`load_data.py`), keep it self-contained and
dependency-light. Prefer standard scientific Python (NumPy, SciPy, Matplotlib,
h5py).

---

## Reporting issues

Open a GitHub issue to report missing data, incorrect metadata, or broken
file links.
